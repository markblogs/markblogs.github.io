Redis-基础2-String命令

发表于 2020-03-24 | 分类于 Redis

设置key值

命令用于设置给定 key 的值。如果 key 已经存储其他值， SET 就覆写旧值，且无视类型。

命令:SET KEY_NAME VALUE
返回值

在 Redis 2.6.12 以前版本， SET 命令总是返回 OK 。
从 Redis 2.6.12 版本开始， SET 在设置操作成功完成时，才返回 OK 。

实例

1）对不存在的键进行设置

2)对已存在的键进行设置

获取Key值

用于获取指定 key 的值。如果 key 不存在，返回 nil 。如果key 储存的值不是字符串类型，返回一个错误。

命令:GET KEY_NAME
返回值

​ 返回 key 的值，如果 key 不存在时，返回 nil。 如果 key 不是字符串类型，那么返回一个错误。

实例

1）对不存在的 key 或字符串类型 key 进行 GET

2)对不是字符串类型的 key 进行 GET

GETRANGE key start end (字符串值的子字符)

用于获取存储在指定 key 中字符串的子字符串。字符串的截取范围由 start 和 end 两个偏移量决定(包括 start 和 end 在内)。

命令：GETRANGE KEY_NAME start end
返回值

​ 截取得到的子字符串。

实例

GETSET key value(值设为 value ，并返回 key 的旧值)

用于对 key 所储存的字符串值，获取指定偏移量上的位(bit)。

命令:GETBIT KEY_NAME OFFSET
返回值

字符串值指定偏移量上的位(bit)。
当偏移量 OFFSET 比字符串值的长度大，或者 key 不存在时，返回 0

GETBIT key offset(对 key 所储存的字符串值，获取指定偏移量上的位)

用于对 key 所储存的字符串值，获取指定偏移量上的位(bit)。

命令:GETBIT KEY_NAME OFFSET
返回值

字符串值指定偏移量上的位(bit)。
当偏移量 OFFSET 比字符串值的长度大，或者 key 不存在时，返回 0 。

MGET key1 [key2..] (所有(一个或多个)给定 key 的值)

返回所有(一个或多个)给定 key 的值。 如果给定的 key 里面，有某个 key 不存在，那么这个 key 返回特殊值 nil 。

命令: MGET KEY1 KEY2 .. KEYN
返回值

一个包含所有给定 key 的值的列表。

实例

SETBIT key offset value(设置或清除指定偏移量上的位(bit))

对 key 所储存的字符串值，设置或清除指定偏移量上的位(bit)。

命令:Setbit KEY_NAME OFFSET
返回值

指定偏移量原来储存的位。

SETEX key seconds value(设置过期时间，秒)

为指定的 key 设置值及其过期时间。如果 key 已经存在， SETEX 命令将会替换旧的值。
命令：SETEX KEY_NAME TIMEOUT VALUE

返回值

设置成功时返回 OK 。

实例

SETNX key value(key 不存在时设置 key 的值)

命令在指定的 key 不存在时，为 key 设置指定的值。

命令: SETNX KEY_NAME VALUE
返回值

设置成功，返回 1 。 设置失败，返回 0 。

实例

SETRANGE key offset value(用 value 参数覆写给定 key 所储存的字符串值，从偏移量 offset 开始。)

命令用指定的字符串覆盖给定 key 所储存的字符串值，覆盖的位置从偏移量 offset 开始。

命令：SETRANGE KEY_NAME OFFSET VALUE
返回值

被修改后的字符串长度。

实例

STRLEN key(key 所储存的字符串值的长度)

用于获取指定 key 所储存的字符串值的长度。当 key 储存的不是字符串值时，返回一个错误。

命令：STRLEN KEY_NAME
返回值

字符串值的长度。 当 key 不存在时，返回 0。

实例

获取字符串的长度

MSET key value key value . . . (同时设置一个或多个 key-value 对)

命令用于同时设置一个或多个 key-value 对。

命令：MSET key1 value1 key2 value2 .. keyN valueN
返回值

总是返回 OK 。

实例

MSETNX key value key value …(同时设置一个或多个对，当且仅当所有给定 key 都不存在。)

命令用于同时设置一个或多个 key-value 对。

命令：MSETNX key1 value1 key2 value2 .. keyN valueN
返回值

当所有 key 都成功设置，返回 1 。 如果所有给定 key 都设置失败(至少有一个 key 已经存在)，那么返回 0 。

实例

PSETEX key milliseconds value(设置过期时间，毫秒)

命令以毫秒为单位设置 key 的生存时间。

命令：PSETEX key1 EXPIRY_IN_MILLISECONDS value1
返回值

设置成功时返回 OK 。

实例

INCR key(将 key 中储存的数字值增一。)

Incr 命令将 key 中储存的数字值增一。
如果 key 不存在，那么 key 的值会先被初始化为 0 ，然后再执行 INCR 操作。
如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误。
本操作的值限制在 64 位(bit)有符号数字表示之内。

命令：INCR KEY_NAME
返回值

执行 INCR 命令之后 key 的值。

INCRBY key increment（储存的值加上给定的增量值）

Incrby 命令将 key 中储存的数字加上指定的增量值。
如果 key 不存在，那么 key 的值会先被初始化为 0 ，然后再执行 INCRBY 命令。
如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误。
本操作的值限制在 64 位(bit)有符号数字表示之内。

命令：INCRBY KEY_NAME INCR_AMOUNT
返回值

加上指定的增量值之后， key 的值。

INCRBYFLOAT key increment(值加上给定的浮点增量值)

为 key 中所储存的值加上指定的浮点数增量值。
如果 key 不存在，那么 INCRBYFLOAT 会先将 key 的值设为 0 ，再执行加法操作。

命令：INCRBYFLOAT KEY_NAME INCR_AMOUNT
返回值

执行命令之后 key 的值。

实例

DECR key(将 key 中储存的数字值减一)

命令将 key 中储存的数字值减一。
如果 key 不存在，那么 key 的值会先被初始化为 0 ，然后再执行 DECR 操作。
如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误。
本操作的值限制在 64 位(bit)有符号数字表示之内。

命令：DECR KEY_NAME
返回值

执行命令之后 key 的值。

实例

DECRBY key decrement(值减去给定的减量值)

命令将 key 所储存的值减去指定的减量值。
如果 key 不存在，那么 key 的值会先被初始化为 0 ，然后再执行 DECRBY 操作。
如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误。
本操作的值限制在 64 位(bit)有符号数字表示之内。

命令：DECRBY KEY_NAME DECREMENT_AMOUNT
返回值

减去指定减量值之后， key 的值。

APPEND key value(在原来值后添加字符串)

用于为指定的 key 追加值。
如果 key 已经存在并且是一个字符串， APPEND 命令将 value 追加到 key 原来的值的末尾。
如果 key 不存在， APPEND 就简单地将给定 key 设为 value ，就像执行 SET key value 一样。

命令: APPEND KEY_NAME NEW_VALUE
返回值

追加指定值之后， key 中字符串的长度。

实例
