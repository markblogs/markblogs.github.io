创建 shell 脚本

创建 shell 文件，必须在文件第一行使用指定的 shell：

执行shell脚本

执行 shell 脚本首先要查看 用户是否具有执行文件(需要执行的shell文件) 文件权限

输出环境变量

环境变量配置在 /etc/profile

在 shell 脚本中可以使用 $ + 变量名 进行环境变量查看和使用

用户变量

shell脚本中用户自定义的变量，用户变量长度不可以超过 20 位

用户变量在使用时 $ + 变量名 进行使用

命令替换

从命令中提取信息，并将其赋值给变量。

有两种方式可以将命令赋值给变量

  • 反引号字符 (`)
  • $() 格式

重定向输入和输出

重定向有两种

  • 将命令输出重定向到另一个位置：输出重定向

    • > 重定向命令，会覆盖文件原有内容

    • >> 在原有文件进行追加

  • < 可以将文件重定向到命令输入：输入重定向

执行数学运算

  • expr 命令

  • 使用方框号

  • 浮点值运算 bc 命令

退出脚本

shell 中 每运行一个命令都使用退出状态码 （exit status）

Linux 专门提供了变量 $? 变量来保存上一个命令的退出状态码。

exit 命令 允许在脚本结束时指定一个状态码

Jenkins-配置2-集成github

Posted on 2020-07-26 Edited on 2021-04-23 In 持续集成 Jenkins

集成github

jenkins配置
  • Manage Jenkins -》 Configure System -》 Github 服务器
    • 配置 Secret text 凭证() ，Secret text 生成方式 Github 用户设置 -》settings -》Developer settings -》Personal access token 将生成好的token复制作为 Secret text
  • 配置github服务器凭据 ，上一步生成的凭据
  • 高级重写 Hook URL
  • 进入 我们的工程配置 构建触发器节点 选择 GitHub hook trigger for GITScm polling

github配置
  • 生成token Github 用户设置 -》settings -》Developer settings -》Personal access token 作为 jenkins 的Secret text 凭证()
    • 需要勾选 repo、admin:repo_hook 两个节点
  • 进入 仓库 -》 settings -》Webhooks
    • 设置 Payload URL
    • Payload URL 为jenkins 在 高级重写 Hook URL 设置的地址

命令

常用命令

  • -maven compile（编译）：将java源程序编译成class字节文件，target目录下

  • -maven test（测试）：运行src/test/java下的测试类

  • -maven clean (清除)  ：将以前编译好的class字节码文件删除

  • -maven package (打包) ：打到target 目录下

  • -maven install ：将项目生成的jar包，放到仓库中，以便别的模块使用。

  • 分析项目的依赖信息：mvn dependency:analyze 或 mvn dependency:tree
