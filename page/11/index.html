鼠标事件

  1. mousedown

    鼠标在指定元素上被按下就会被触发。

  2. mouseup

    鼠标指针在元素上并释放鼠标键，触发

  3. mouseover

    鼠标指针进入指定元素的区域(包含子元素)，触发

  4. mouseout

    鼠标指针离开指定元素的区域，触发

  5. mouseenter

    当鼠标指针穿过指定（进入）被选元素时，会发生 mouseenter 事件。

    注意：与 mouseover 事件不同，mouseenter 事件只有在鼠标指针进入被选元素时被触发，mouseover 事件在鼠标指针进入任意子元素时也会被触发

  6. mouseleave

    当鼠标指针离开被选元素时，会发生 mouseleave 事件。

    注意：与 mouseout 事件不同，mouseleave 事件只有在鼠标指针离开被选元素时被触发，mouseout 事件在鼠标指针离开任意子元素时也会被触发。

入口函数

jQuery 入口函数与 JavaScript 入口函数的区别：

  • jQuery 的入口函数是在 html 所有标签(DOM)都加载之后，就会去执行。
  • JavaScript 的 window.onload 事件是等到所有内容，包括外部图片之类的文件加载完后，才会执行。

load 和 ready 区别

window.onload $(document).ready()
执行时机 必须等待网页全部加载完毕（包括图片等），然后再执行包裹代码 只需要等待网页中的DOM接口加载完毕，就能执行包裹代码
执行次数 只能执行一次，如果第二次，那么第一次的执行会被覆盖 可以执行多次，第N此都不会被上一次覆盖

Docker_基础3_基础命令

Posted on 2020-06-27 Edited on 2021-04-23 In 容器技术

帮助命令

  • 查看版本：
    • docker version
  • docker信息：
    • docker info
  • docker命令帮助：
    • docker –help

镜像命令

  • 列出本机所有镜像：
    • docker images [options]
      • -a：列出本地所有的镜像（含中间映像层）
      • -q：只显示镜像ID
      • –digests：显示镜像的摘要信息
      • –no-trunc：显示完整的镜像信息
  • 仓库中检索镜像：
    • docker search [options]
      • –no-trunc：显示完整镜像描述
      • -s：列出收藏数不小于指定值的镜像
      • –automated：只列出 automated build（自动构建）类型的镜像
  • 拉取镜像：
    • docker pull ImageName:tag
  • 移除镜像：
    • docker rmi ImageName
    • 移除所有镜像：docker rmi -f $(docker images -qa)

容器命令

  • 新建并启动容器

    • docker run [options] image [command]

      • –name：容器名称
      • -d：后台运行，并返回容器id
      • -i：交互模式运行容器，通常与 -t 同时使用
      • -t：为容器重新分配一个伪输入终断
      • -P：随机端口映射
      • -p：指定端口映射

  • 查看所有正在运行的容器

    • docker ps

  • 退出容器

    • exit：停止并退出容器
    • ctrl+p+q：容器不停止退出

  • 启动容器

    • docker start 容器id或者容器名

  • 重启容器

    • docker restart 容器id或者容器名

  • 停止容器

    • docker stop 容器id或者容器名

  • 强制停止容器

    • docker kill 容器id或者容器名

  • 删除已经停止的容器

    • docker rm 容器id
    • docker rm $(docker ps -aq)

  • 查看容器日志

    • docker logs -f -t –tail 容器id
      • -t：加入时间戳
      • -f：跟随最新的日志打印
      • –tail 数字：显示最后多少条

  • 查看容器内运行的进程

    • docker top 容器id

  • 查看容器内部细节（）

    • docker inspect 容器id

  • 进入正在运行的容器并以命令交互

    • docker exec -it 容器id bashShell （容器中打开新的终端，不会的启动新的进程）
    • 重新进入：docker attch ring器id（直接进入容器启动命令的终端，不会启动新的进程）

  • 从容器内拷贝文件到主机上

    • docker cp 容器id:容器内路径 目的主机路径

Docker_基础1_简介

Posted on 2020-06-27 Edited on 2021-04-23 In 容器技术

docker简介

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的镜像中，然后发布到任何流行的 Linux或Windows机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口

​ docker理念：一次封装到处运行

​ docker包含两方面技术 ：镜像技术、容器虚拟技术

docker优势

特性 容器 虚拟机
启动 秒级 分钟
磁盘使用 一般为 MB 一般为 GB
性能 接近原生 弱于
系统支持量 单机支持上千个容器 一般为几十个
  • 更高效的利用资源：docker容器不需要像虚拟机运行完整的系统造成额外的开销。
  • 更快速的启动时间：对比虚拟机的服务，docker容器启动更快。
  • 一致的运行环境：通过镜像技术，可以达到 开发、测试、生产等环境的一致性

docker三要素

  • 镜像（Images）
  • 容器（container）
  • 仓库（repository）

CompletableFuture

​ CompleteableFuture 是 Future 的扩展

CompletableFuture 方法

  • 构造方法
    • public CompletableFuture()
  • 实例化 CompletableFuture
    • 运行一个异步任务，不需要任务返回任何结果
      • CompletableFuture.runAsync(Runnable runnable);
      • CompletableFuture.runAsync(Runnable runnable, Executor executor);
    • 运行一个异步任务，需要任务返回执行结果
      • CompletableFuture.supplyAsync(Supplier supplier);
      • CompletableFuture.supplyAsync(Supplier supplier, Executor executor)
  • 运行 supplyAsync 任务，获取返回结果
    • get()
  • CompletableFuture 转换和运行
    • thenApply()：对返回结果进行操作
    • thenAccept() ：运行结束后，运行一个回调方法，可以访问CompletableFuture结果
    • thenRun()：运行结束后，运行一个回调方法，不可以访问CompletableFuture结果
  • 组合两个 CompletableFuture
    • thenCompose()：组合两个独立的 CompletableFuture，获取合并后的结果。
    • thenCombine()：组合两个独立的 CompletableFuture，取得两个结果集后进行操作
  • 组合多个 CompletableFuture
    • allof() ：组合所有的CompletableFuture都结束后，完成一些事情
    • anyof()：组合所有的CompletableFuture有一个完成后，就返回一个新的CompletableFuture
  • 异常处理
    • exceptionally 回调处理异常
    • handle()：无论异常是否发生都会被调用

Java异步-基础1-Future

Posted on 2020-06-21 Edited on 2021-04-23 In Java异步

什么是Future

​ 它建模了一种异步计算，返回一个执行运算结果的引用，当运算结果结束后，这个引用被返回给调用方。在Future中触发那些潜在耗时的操作把调用线程解放出来。在处理异步调用和并发处理时非常有用。

局限性
  • 不能将两个异步操作合并为一个（两个异步相互独立，同时第二个又依赖第一个结果）
  • 等待Future集合中所有任务都完成
  • 仅等待Future集合中最开结束的任务完成，并返回它的结果

Future可以做什么？

  • 异步计算
  • 操作大数据结构
  • 远程方法调用(下载文件、抓取HTML、Web服务)

Future 方法

  • get():方法可以当任务结束放回一个结果，如果调用时，工作还没有结束，则会阻塞线程，知道任务执行完成。
  • get(long timeout,TimeUnit unit) ：最多等待timeout时间，返回结果
  • cancel(boolean mayInterruptIfRunning)： 方法可以用来停止一个任务，如果任务可以停止（通过mayInterruptIfRunning判断），则可以返回true，如果任务已经完成或者已经停止，或者这个任务无法停止，则会返回false。
  • isDone()：方法判断当前方法是否完成
  • icCancel()：方法判断当前方法是否取消
