Queue

​ 队列的方法接口，包含入队、出队、检索

方法
返回值 方法 作用
boolean add(E e) 将指定的元素插入到此队列中，如果可以立即执行此操作，而不会违反容量限制， true在成功后返回 IllegalStateException如果当前没有可用空间，则抛出IllegalStateException。
E element 检索，但不删除，这个队列的头。
boolean offer(E e) 如果在不违反容量限制的情况下立即执行，则将指定的元素插入到此队列中。
E peek() 检索但不删除此队列的头，如果此队列为空，则返回 null 。
E pool() 检索并删除此队列的头，如果此队列为空，则返回 null 。
E remove() 检索并删除此队列的头。

BlockingQueue（单向队列）

BlockingQueue即阻塞队列，它是基于ReentrantLock；是所有阻塞队列父类接口，定义了所有队列的阻塞方法。

BlockingQueue方法有四种形式，具有不同的操作方式：

  • 抛出异常
  • 返回一个特殊值（null 或 false）
  • 程序将无限期的阻止当前线程，直到操作成功为止
  • 程序块在指定最大实现后放弃
出队入队方法
操作 立即返回（true/false） 立即返回（true/抛出异常） 一直等待直到成功 等待最大时限
添加 offer(E e) add(E e) put(E e) offer(E e, long timeout, TimeUnit unit)
移除 pool() remove() take() poll(long timeout, TimeUnit unit)
其他方法
返回值 方法 作用
boolean contains(Object o) 如果此队列包含指定元素，则返回true
int drainTo(Collection<? super E> c) 从该队列中删除所有可用的元素，并将它们添加到给定的集合中
int drainTo(Collection<? super E> c) 最多从该队列中删除给定数量的可用元素，并将它们添加到给定集合中
BlockingQueue实现类
  • LinkedBlockingQueue
    • 使用双锁保证线程安全
    • 采取双锁，所以读写操作可以并行执行
    • 内部实现为链表
  • ArrayBlockingQueue
    • 使用单锁保证线程安全
    • 采取单锁，读写操作异步进行，不能并行执行
    • 内部实现为动态数组
    • 因为使用单锁机制，所以高并发情况下LinkedBlockingQueue性能优于 ArrayBlockingQueue
    • remove 时需要移动数据，影响性能

第 1 位

-： 普通文件

插入排序

  • 将n个待排序元素，看成一个有序表和一个无序表，

  • 对比无序列表第一个元素和有序列表最后一个元素大小，

  • 如果无序列表第一个元素小于有序列表最后一个元素

  • 将无序列表的第一个元素依次和有序列表（从大到小）对比

  • 如果小于有序列表元素那么相互替换位置，继续向前和有序列表元素进行对比

  • 否则直接赋值到对比的有序列表位置
