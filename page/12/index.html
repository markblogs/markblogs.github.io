实现类

  • ArrayList
    • 实现List接口
    • 内部实现为Object[]动态数组
    • 对于随机访问get和set，ArrayList觉得优于LinkedList，因为LinkedList要移动指针
    • 不是线程安全的
  • LinkedList
    • 双链表实现了List和Deque接口，可以被当做堆栈、队列、双端队列进行操作
    • 内部实现为链表
    • 不是线程安全的
    • 对于新增和删除操作add和remove，LinedList比较占优势，因为ArrayList要移动数据。
    • 顺序访问非常高效
  • Vector
    • 实现List接口
    • Vector的数据结构和ArrayList差不多，内部都为Object[]动态数组
    • 是线程安全的，通过重载大量的父类方法加入synchronized 修饰来达到线程安全效果
  • Stack
    • 由于Stack继承于Vector，因此它包含Vector中的全部API。
    • 多了栈操作：pop()、peek()、push(E object)
  • CopyOnWriteArrayList
    • 线程安全的，保证了数据最终一致，但是不能做到数据实时一致
    • 所有写入操作进行加锁，读操作没有加锁
    • Vector 增删查改都加入了synchronized；CopyOnWriteArrayList 增删改加入了锁，读没有加锁，所以在性能上优于Vector
    • 内存问题，在修改时存在两个对象。占用两个内存，如果修改操作过多会导致频繁的GC

使用总结

  • 对于需要快速插入，删除元素，应使用ArrayList
  • 对于需要快速随机访问元素，应该使用LinkedList
  • 对于 单线程 或者多线程环境，但List只会被单个线程操作，此时应该使用非同步的类（ArrayList）
  • 对于 多线程环境，且List可能是被多个线程操作，此时，应该使用同步类（Vector）

BlockingQueue（单向队列）

BlockingQueue即阻塞队列，它是基于ReentrantLock；是所有阻塞队列父类接口，定义了所有队列的阻塞方法。

BlockingQueue方法有四种形式，具有不同的操作方式：

  • 抛出异常
  • 返回一个特殊值（null 或 false）
  • 程序将无限期的阻止当前线程，直到操作成功为止
  • 程序块在指定最大实现后放弃

BlockingQueue实现类
  • LinkedBlockingQueue
    • 使用双锁保证线程安全
    • 采取双锁，所以读写操作可以并行执行
    • 内部实现为链表
  • ArrayBlockingQueue
    • 使用单锁保证线程安全
    • 采取单锁，读写操作异步进行，不能并行执行
    • 内部实现为动态数组
    • 因为使用单锁机制，所以高并发情况下LinkedBlockingQueue性能优于 ArrayBlockingQueue
    • remove 时需要移动数据，影响性能
